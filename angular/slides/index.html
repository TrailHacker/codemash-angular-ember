Modules – Container for logically grouping components

Services/Factories/Providers – Reusable logic components

Modules

Container grouping components under a common name

AngularJS is bootstrapped with a starting module data-ng-app='someModule'

Promotes module based layout

AngularJS docs recommend (from https://docs.angularjs.org/guide/module) A module for each feature A module for each reusable component (especially directives and filters) And an application level module which depends on the above modules and contains any initialization code.

Module Syntax

  • angular.module('moduleName', []);
  • angular.module('moduleName');

With [] defines a module, and running that statement a second time will overwrite the first including any components attached to it.

Without [] gets a reference to the module named so components can be attached to it.

Scope

  • The glue where the model is stored
  • Items on the scope are visible to the view and the controller
  • Isolates controllers from the DOM and directives

$rootScope is the parent, child scopes are created from $rootScope

Scopes inherit the properties, etc from their parents

Template

  • The HTML containing AngularJS components
  • Templates show data using expressions
  • Expressions look like {{ expression }} and resolve to models attached to the scope

Filter

  • Allows formatting model data for display
  • Used like {expression | filter}
  • Can be chained {expression | filter1 | filter 2}

Chained expressions are processed in order left to right

Directive

  • How AngularJS attaches behavior to the DOM
  • Notation can be ng-thing, data-ng-thing, x-ng-thing
  • AngularJS has many built in, but you can create your own

Controller

  • Add state/models and functions to scope
  • Should
    • Setup initial scope model values
    • Add behavior related to their scope/models
  • Should Not
    • Manipulate the DOM (use directives)
    • Format model values (use filters)
    • Be used to share logic (use services/factories/providers)

Service/Factory/Provider

  • Service - The result of calling new Service()
  • Factory - The result of executing Factory()
  • Provider - Can be configured in a module config
  • Values/Constants - Values can’t be used in module config

All very similar and can often be used interchangably, the above lists the major differences

All are singletons

Service/Factory/Provider Continued...

  • Encapsulate business logic
  • Share logic, state across the application
  • AngularJS provides several, but you can create your own
